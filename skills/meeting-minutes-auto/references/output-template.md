# 统一纪要 Markdown 输出模板

所有平台的纪要拉取后，统一按此模板格式化，再写入腾讯文档。

## 模板内容

```markdown
# {会议主题} · 会议纪要

> 平台：{腾讯会议/飞书/钉钉} ｜ 生成时间：{YYYY-MM-DD HH:MM}
> 纪要来源：{平台原生AI / Python脚本}

---

|------|------|
| 会议主题 | {主题} |

| 会议平台 | {腾讯会议/飞书/钉钉} |
| 参会人数 | {N}人 |
| 主要参会人 | {姓名1}、{姓名2}、{姓名3}... |
| 录制方式 | {云录制/妙记/听记} |

---

## AI 摘要

{平台原生AI生成的会议总结，直接引用，保持原汁原味}

---
```

## 字段映射说明

不同平台的数据映射到模板字段：

### 腾讯会议 (tmeet)
- 会议信息 ← meeting list-ended + report participants
- AI 摘要 ← record smart-minutes
- 议题与结论 ← smart-minutes 中的议题和结论
- 待办事项 ← smart-minutes 中的行动项
- 完整转写 ← record transcript-get

### 飞书 (feishu)
- 会议信息 ← vc +detail
- AI 摘要 ← minutes --summary
- 议题与结论 ← minutes --chapter（章节分段）
- 待办事项 ← minutes --todo
- 完整转写 ← minutes --transcript

### 钉钉 (dingtalk)
- 会议信息 ← minutes get info
- AI 摘要 ← minutes get summary
- 议题与结论 ← summary 中的讨论点
- 待办事项 ← minutes get todos
- 关键关键词 ← minutes get keywords
- 完整转写 ← minutes get transcription

### Python 脚本
- 会议信息 ← 脚本输出的技术信息表
- 议题与结论 ← DeepSeek 结构化总结
- 待办事项 ← DeepSeek 提取的待办
- 完整转写 ← Whisper 转写文本

## 格式化注意事项

1. **时间格式统一**：转为 `YYYY-MM-DD HH:MM` 格式
2. **发言人统一**：尽量用真实姓名，不用"发言人1"
3. **待办事项**：如果没有明确截止时间，填"-"；优先级根据上下文推断
4. **转写文本**：保留时间戳，按发言人分段，过长可折叠
5. **AI 摘要**：保持平台原生输出，不二次加工（保留原汁原味）
